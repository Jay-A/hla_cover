hla_solver.selectors module¶

selectors.py

This module provides functions to identify top-performing alleles or sample combinations that maximize dataset coverage. These tools are useful for initializing or benchmarking HLA greedy solvers.

Author: Jay M. Appleton

hla_solver.selectors.analyze_hla_group(df: DataFrame, group_name: str, col1: str, col2: str, timing: bool = False, required_allele_filter: List[Tuple[str, str, str]] | None = None, relative_to_full_df: bool = False) → DataFrame | None[source]¶

Analyze allele frequency and sample coverage for a specific HLA group.

This function evaluates the occurrence and coverage of unique alleles from the two specified columns (representing a single HLA group), optionally filtered by required allele presence in other columns.

Note

This function is currently included in the selectors module, but may be better suited in a future analysis module.

Parameters:

  • df (pandas.DataFrame) – Input DataFrame containing HLA allele columns.

  • group_name (str) – Human-readable name for the group (e.g., “HLA-A”).

  • col1 (str) – First column containing alleles for this group.

  • col2 (str) – Second column containing alleles for this group.

  • timing (bool, optional) – If True, prints timing information (default: False).

  • required_allele_filter (list of tuple(str, str, str), optional) – List of required allele filters applied before analysis. Each tuple is (col1, col2, allele).

  • relative_to_full_df (bool, optional) – If True, coverage percentages are calculated relative to the original (unfiltered) dataset. If False, percentages are relative to the filtered subset (default: False).

Returns:

DataFrame with allele statistics sorted by matched sample count. If no data remains after filtering, returns None.

Columns:

  • ”Allele ID”: Internal numeric ID

  • ”Allele Name”: Original allele string

  • ”Matched Sample Count”: Number of samples containing allele

  • ”Percentage Cover”: Coverage as percentage (float)

Return type:

pandas.DataFrame or None

Examples

>>> analyze_hla_group(df, "HLA-A", "HLA-A1", "HLA-A2")
hla_solver.selectors.find_best_single_allele_filter(df, hla_groups, timing=False)[source]¶

Identify the best single HLA allele (from any group) to use as a filter based on its ability to filter the most samples from the dataset.

This function scans the top allele in each HLA group and evaluates its filtering impact, returning the one that filters the largest number of samples.

Parameters:

  • df (pd.DataFrame) – Full dataset of samples with HLA allele columns.

  • hla_groups (list of tuples) – List of (group_name, col1, col2) HLA group definitions.

  • timing (bool, optional) – Whether to print timing/debug info. Defaults to False.

Returns:

  • best_filter (tuple): (col1, col2, allele) representing the most impactful filter.

  • best_stats (dict): Dictionary summarizing filter impact (sample count, etc.).

Return type:

tuple

hla_solver.selectors.find_matching_samples_by_filter(df: DataFrame, filter_alleles: Dict[str, List[str]]) → DataFrame[source]¶

Identifies samples that fully match a given set of allele filters for each HLA group.

Parameters:

  • df (pandas.DataFrame) – DataFrame containing HLA allele data.

  • filter_alleles (dict) – Dictionary specifying allele pairs to match per HLA group.

Returns:

Subset of the original DataFrame with samples that match all specified filters.

Return type:

pandas.DataFrame

Raises:

ValueError – If the expected columns for an HLA group are not found.

Notes

  • Matching is done regardless of allele order (allele1, allele2) vs (allele2, allele1).

  • Column names are matched using group prefix (e.g. “HLA-A”, “HLA-B”).

Examples

>>> filter_alleles = {
...     "HLA-A": ["02:01", "24:02:04"],
...     "HLA-B": ["07:02", "15:01"]
... }
>>> matched_df = find_matching_samples_by_filter(df, filter_alleles)
hla_solver.selectors.find_top_covering_samples(df: DataFrame, hla_groups: List[Tuple[str, str, str]], top_n: int, allele_to_id: Dict[str, int], allele_matrix: ndarray, allele_id_to_positions: Dict[int, Set[int]], total_samples: int | None = None, current_subset_indices: Set[int] | None = None) → List[Dict[str, Any]][source]¶

Identifies the top N samples with the highest dataset coverage based on their alleles, using precomputed allele mappings.

Parameters:

  • df (pandas.DataFrame) – Subset of the full HLA dataset (used for evaluation).

  • hla_groups (list of tuple(str, str, str)) – Each tuple defines an HLA group as (group_name, col1, col2).

  • top_n (int) – Number of top-covering samples to return.

  • allele_to_id (dict) – Mapping from allele string to unique integer ID.

  • allele_matrix (numpy.ndarray) – Matrix of allele IDs (shape: num_samples x num_allele_columns).

  • allele_id_to_positions (dict) – Maps allele ID to set of sample indices in the full dataset.

  • total_samples (int, optional) – Total number of samples in the original dataset (used for % calc). Defaults to length of df if not provided.

  • current_subset_indices (set of int, optional) – Indices (in original dataset) corresponding to the current df rows.

Returns:

List of top samples with allele info and match coverage metrics.

Return type:

list of dict

Notes

This function assumes: - Alleles in df are already cleaned/stripped - allele_matrix rows correspond 1:1 with df rows

hla_solver.selectors.find_top_filter_combinations(df: DataFrame, hla_groups: List[Tuple[str, str, str]], top_k_per_group: int = 5, top_n_results: int = 8, timing: bool = False) → List[Dict[str, Any]][source]¶

Finds the top combinations of single allele filters (one per group) that result in the highest dataset coverage.

This function identifies the most common alleles within each HLA group, then exhaustively tests all possible one-allele-per-group filter combinations to determine which combinations match the most samples.

Parameters:

  • df (pandas.DataFrame) – Input dataset containing allele columns.

  • hla_groups (list of tuple(str, str, str)) – Each tuple defines an HLA group as (group_name, col1, col2).

  • top_k_per_group (int, optional) – Number of top alleles to select from each group (default: 5).

  • top_n_results (int, optional) – Number of top filter combinations to return (default: 8).

  • timing (bool, optional) – If True, enables timing and verbose analysis output (default: False).

Returns:

Each result dictionary contains:

  • ”Combo”: list of (group_name, col1, col2, allele)

  • ”Matched Samples”: int

  • ”Coverage (%)”: float

Return type:

list of dict

Example

>>> find_top_filter_combinations(df, hla_groups=[("HLA-A", "HLA-A1", "HLA-A2")])
hla_solver.selectors.multi_sample_coverage_and_remainder(df: DataFrame, hla_groups: List[Tuple[str, str, str]], sample_ids: List[str], match_cache: Dict[Tuple[str, str], Series] | None = None) → Tuple[float | None, Dict[str, set] | None, Dict[Tuple[str, str], Series], DataFrame | None][source]¶

Calculates dataset coverage based on alleles from selected samples and returns the DataFrame of unmatched (uncovered) samples.

Parameters:

  • df (pandas.DataFrame) – DataFrame containing HLA allele columns.

  • hla_groups (list of tuple) – List of (group_name, col1, col2) tuples specifying HLA groups and their columns.

  • sample_ids (list of str) – Sample IDs (from ‘Sample ID’ column) to be used for allele collection.

  • match_cache (dict, optional) – Cache mapping (group, allele) -> boolean Series of matches. Improves performance for repeated calls.

Returns:

  • coverage_pct (float or None) – Percentage of dataset covered by the alleles from selected samples.

  • combined_allele_dict (dict or None) – Dictionary of HLA group to set of alleles used for filtering. Format: {group_name: set of alleles}

  • updated_cache (dict) – Updated match cache.

  • uncovered_df (pandas.DataFrame or None) – DataFrame of unmatched samples (i.e., samples not covered by any selected alleles).

hla_solver.selectors.multi_sample_coverage_with_cache(df: DataFrame, hla_groups: List[Tuple[str, str, str]], sample_indices: List[int], match_cache: Dict[Tuple[str, str], Series] | None = None) → Tuple[float | None, Dict[str, set] | None, Dict[Tuple[str, str], Series]][source]¶

Computes overall dataset coverage for a given list of sample indices, using combined allele filters across multiple HLA groups. Optionally caches match results to accelerate repeated evaluations.

Parameters:

  • df (pandas.DataFrame) – DataFrame containing HLA allele columns.

  • hla_groups (list of tuple) – List of (group_name, col1, col2) specifying the HLA groups and their columns.

  • sample_indices (list of int) – Row indices in df representing the selected samples.

  • match_cache (dict, optional) – Dictionary to store or reuse cached Series matching each (group, allele) combination.

Returns:

  • coverage_pct (float or None) – Percentage of dataset matched by the combined allele set. Returns None if inputs are invalid.

  • combined_allele_dict (dict or None) – Dictionary of alleles used for filtering, grouped by HLA group. Format: {group_name: set of alleles}

  • updated_cache (dict) – Updated cache containing match Series for each (group, allele).

Notes

  • Matching is based on OR logic across alleles within a group.

  • Cached results avoid repeated string comparisons for the same allele.

hla_solver.selectors.sample_coverage_percent(df: DataFrame, hla_groups: List[Tuple[str, str, str]], sample_id: str, coverage_type: str = 'union', verbose: bool = False) → Tuple[float | None, Dict[str, List[str]] | None][source]¶

Calculate the percentage of the dataset that shares alleles with a given sample, identified by ‘Sample ID’, either by union (any matching allele across any group) or intersection (all alleles matched, group-wise).

Parameters:

  • df (pandas.DataFrame) – The dataset containing HLA allele information.

  • hla_groups (list of tuple) – List of tuples specifying HLA group name and its two column names. Example: [(“HLA-A”, “HLA-A_1”, “HLA-A_2”), …]

  • sample_id (str) – The unique Sample ID to evaluate from the DataFrame.

  • coverage_type (str, optional) – Type of coverage to compute: “union” (default) or “intersection”.

  • verbose (bool, optional) – Whether to print detailed output. Default is False.

Returns:

  • coverage_percentage (float or None) – Percentage of the dataset matching the given coverage logic. Returns None if the Sample ID is not found.

  • allele_dict (dict or None) – Dictionary mapping group name to the list of the sample’s alleles. Returns None if the Sample ID is not found.
