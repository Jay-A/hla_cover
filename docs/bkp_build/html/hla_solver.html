hla_solver package¶

Provides an HLA greedy coverage solver with parallelization support.

Modules: - solver: Greedy HLA coverage algorithm - backend: Parallelization strategies (serial, thread pool, process pool) - utils: Logging and helper utilities - preprocessing: Dataset parsing and preparation - validation: Coverage scoring and testing tools - selectors: Experimental filtering and sample-selection strategies

class hla_solver.BackendManager(mode='serial', num_workers=1, verbose=False, progress_mode='progress_bar')[source]¶

Bases: object

Manages backend parallelism strategies for the HLA greedy solver.

Supports three modes:

  • serial: runs everything sequentially.

  • thread_pool: uses a ThreadPoolExecutor for parallelism.

  • process_pool: uses a ProcessPoolExecutor, with warnings if used in Jupyter notebooks.

Parameters:

  • mode (str, optional) – The parallelism mode to use. Must be one of ‘serial’, ‘thread_pool’, or ‘process_pool’. Default is ‘serial’.

  • num_workers (int, optional) – Number of worker threads or processes to use in parallel modes. Default is 1.

  • verbose (bool, optional) – If True, prints progress and debug information. Default is False.

  • progress_mode (str, optional) – Controls progress reporting mode. Options: ‘progress_bar’, ‘per_worker’, ‘silent’. Default is ‘progress_bar’.

Examples

Create a backend manager with 4 threads and verbose output:

>>> backend = BackendManager(mode='thread_pool', num_workers=4, verbose=True)
>>> results = backend.run_local_global_greedy_cover(df, hla_groups, allele_to_id, allele_matrix, allele_id_to_positions)

Run the local-global greedy coverage algorithm using the selected backend.

Dispatches the call to the appropriate internal method based on the mode.

Parameters:

  • df (pandas.DataFrame) – Input data frame containing allele information.

  • hla_groups (list) – Groups of HLA alleles to cover.

  • allele_to_id (dict) – Mapping from allele name to numeric ID.

  • allele_matrix (numpy.ndarray or similar) – Matrix representing alleles and their features.

  • allele_id_to_positions (dict) – Maps allele IDs to positions in the matrix or dataset.

  • max_iter (int, optional) – Maximum iterations for the greedy algorithm. Default is 10.

  • top_n (int, optional) – Number of top solutions to consider in the greedy algorithm. Default is 1.

  • top_k_global (int, optional) – Number of top global solutions to keep. Default is 5.

Returns:

A tuple of (selected_samples, filters, remainder), as returned by the backend algorithm.

Return type:

tuple

Raises:

ValueError – If an unknown backend mode is specified.

hla_solver.find_matching_samples_by_filter(df: DataFrame, filter_alleles: Dict[str, List[str]]) → DataFrame[source]¶

Identifies samples that fully match a given set of allele filters for each HLA group.

Parameters:

  • df (pandas.DataFrame) – DataFrame containing HLA allele data.

  • filter_alleles (dict) – Dictionary specifying allele pairs to match per HLA group.

Returns:

Subset of the original DataFrame with samples that match all specified filters.

Return type:

pandas.DataFrame

Raises:

ValueError – If the expected columns for an HLA group are not found.

Notes

  • Matching is done regardless of allele order (allele1, allele2) vs (allele2, allele1).

  • Column names are matched using group prefix (e.g. “HLA-A”, “HLA-B”).

Examples

>>> filter_alleles = {
...     "HLA-A": ["02:01", "24:02:04"],
...     "HLA-B": ["07:02", "15:01"]
... }
>>> matched_df = find_matching_samples_by_filter(df, filter_alleles)

hla_solver.find_top_filter_combinations(df: DataFrame, hla_groups: List[Tuple[str, str, str]], top_k_per_group: int = 5, top_n_results: int = 8, timing: bool = False) → List[Dict[str, Any]][source]¶

Finds the top combinations of single allele filters (one per group) that result in the highest dataset coverage.

This function identifies the most common alleles within each HLA group, then exhaustively tests all possible one-allele-per-group filter combinations to determine which combinations match the most samples.

Parameters:

  • df (pandas.DataFrame) – Input dataset containing allele columns.

  • hla_groups (list of tuple(str, str, str)) – Each tuple defines an HLA group as (group_name, col1, col2).

  • top_k_per_group (int, optional) – Number of top alleles to select from each group (default: 5).

  • top_n_results (int, optional) – Number of top filter combinations to return (default: 8).

  • timing (bool, optional) – If True, enables timing and verbose analysis output (default: False).

Returns:

Each result dictionary contains:

  • ”Combo”: list of (group_name, col1, col2, allele)

  • ”Matched Samples”: int

  • ”Coverage (%)”: float

Return type:

list of dict

Example

>>> find_top_filter_combinations(df, hla_groups=[("HLA-A", "HLA-A1", "HLA-A2")])
hla_solver.multi_sample_coverage_and_remainder(df: DataFrame, hla_groups: List[Tuple[str, str, str]], sample_ids: List[str], match_cache: Dict[Tuple[str, str], Series] | None = None) → Tuple[float | None, Dict[str, set] | None, Dict[Tuple[str, str], Series], DataFrame | None][source]¶

Calculates dataset coverage based on alleles from selected samples and returns the DataFrame of unmatched (uncovered) samples.

Parameters:

  • df (pandas.DataFrame) – DataFrame containing HLA allele columns.

  • hla_groups (list of tuple) – List of (group_name, col1, col2) tuples specifying HLA groups and their columns.

  • sample_ids (list of str) – Sample IDs (from ‘Sample ID’ column) to be used for allele collection.

  • match_cache (dict, optional) – Cache mapping (group, allele) -> boolean Series of matches. Improves performance for repeated calls.

Returns:

  • coverage_pct (float or None) – Percentage of dataset covered by the alleles from selected samples.

  • combined_allele_dict (dict or None) – Dictionary of HLA group to set of alleles used for filtering. Format: {group_name: set of alleles}

  • updated_cache (dict) – Updated match cache.

  • uncovered_df (pandas.DataFrame or None) – DataFrame of unmatched samples (i.e., samples not covered by any selected alleles).

hla_solver.multi_sample_coverage_with_cache(df: DataFrame, hla_groups: List[Tuple[str, str, str]], sample_indices: List[int], match_cache: Dict[Tuple[str, str], Series] | None = None) → Tuple[float | None, Dict[str, set] | None, Dict[Tuple[str, str], Series]][source]¶

Computes overall dataset coverage for a given list of sample indices, using combined allele filters across multiple HLA groups. Optionally caches match results to accelerate repeated evaluations.

Parameters:

  • df (pandas.DataFrame) – DataFrame containing HLA allele columns.

  • hla_groups (list of tuple) – List of (group_name, col1, col2) specifying the HLA groups and their columns.

  • sample_indices (list of int) – Row indices in df representing the selected samples.

  • match_cache (dict, optional) – Dictionary to store or reuse cached Series matching each (group, allele) combination.

Returns:

  • coverage_pct (float or None) – Percentage of dataset matched by the combined allele set. Returns None if inputs are invalid.

  • combined_allele_dict (dict or None) – Dictionary of alleles used for filtering, grouped by HLA group. Format: {group_name: set of alleles}

  • updated_cache (dict) – Updated cache containing match Series for each (group, allele).

Notes

  • Matching is based on OR logic across alleles within a group.

  • Cached results avoid repeated string comparisons for the same allele.
